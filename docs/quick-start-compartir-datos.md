# Guia rapida para compartir datos seguros

Johaina Chavez, Data Scientist  
Ceache Lab  
Version: 0.1.1  
Fecha: 2026-07-02

Esta guia explica como una persona, comunidad, acopio, organizacion o equipo tecnico puede compartir informacion util sin exponer personas.

## Objetivo

Ayudar a responder preguntas practicas:

- Que se necesita?
- Donde se necesita, a nivel de zona general?
- Que se ofrece?
- En que estado esta?
- Desde cuando se sabe?
- Quien puede verificarlo?
- Que dato no debe circular?

## Lo minimo que sirve

Para que un dato pueda convertirse en evidencia revisable, debe tener:

| Campo | Para que sirve |
|---|---|
| tipo de actor | saber desde que lugar se observa |
| tipo de dato | distinguir necesidad, oferta, acopio, refugio, voluntariado o entrega |
| categoria | agrupar alimentos, agua, medicina, transporte, refugio u otros |
| cantidad y unidad | comparar demanda y oferta |
| zona agregada | ubicar sin exponer direccion sensible |
| estado | saber si es reportado, verificado, en proceso, entregado o cerrado |
| fecha observada | evitar usar datos viejos como si fueran actuales |
| fuente | saber de donde viene |
| permiso de uso | definir si puede analizarse, compartirse o publicarse |

## No enviar

No enviar:

- cedulas, documentos o pasaportes;
- telefonos privados;
- direcciones exactas de personas vulnerables;
- nombres de personas afectadas;
- informacion de salud individual;
- datos de menores;
- personas desaparecidas;
- fotos identificables;
- chats privados completos.

## Formas aceptadas

Se puede compartir informacion por:

- CSV;
- Excel;
- Google Sheet;
- formulario;
- API;
- resumen institucional;
- plantilla de mapeo de campos.

## Como preparar una hoja

1. Usa una fila por observacion.
2. Evita texto libre con datos personales.
3. Usa zonas generales, no direcciones sensibles.
4. Agrega fecha de actualizacion.
5. Indica si el dato fue reportado, verificado o cerrado.
6. Separa necesidades, ofertas, acopios, refugios, voluntariado y entregas.
7. Conserva una columna de fuente.
8. Declara si el dato puede analizarse y si puede publicarse agregado.

## Que pasa despues

Ceache no publica automaticamente la informacion recibida.

Primero se revisa:

- riesgo de informacion personal;
- permiso de uso;
- consistencia minima;
- fecha de actualizacion;
- posibilidad de agregacion segura;
- utilidad para entender coordinacion.

## Frase corta

Comparte datos que permitan coordinar funciones. No compartas datos que permitan identificar personas vulnerables.
